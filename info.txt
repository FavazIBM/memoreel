================================================================================
Context-Aware System
================================================================================

1. .bob_rules - Rules for Bob to Follow
----------------------------------------
This file defines HOW I SHOULD BEHAVE and WHAT PROCESSES I SHOULD FOLLOW:
- Context-aware processing workflow
- How to search and load project context automatically
- Priority order for context files
- Communication preferences
- Code style guidelines

Think of it as: My instruction manual - "Bob, this is how you should work"


2. project_context/ - Project Encyclopedia
-------------------------------------------
This directory contains INFORMATION ABOUT YOUR PROJECT:
- project_overview.md - What the project is about
- architecture.md - How the system is built
- coding_standards.md - How code should be written
- api_documentation.md - API specifications
- business_logic.md - Business rules and workflows
- dependencies.md - Tech stack and tools
- context_index.md - Quick reference guide
- README.md - How to use the context system

Think of it as: Your project's encyclopedia - "This is what the project is and how it works"

================================================================================
SUMMARY
================================================================================

.bob_rules = Instructions for Bob (the AI)
project_context/ = Knowledge about your project

Together, they enable Bob to work intelligently with full understanding of 
your project, following your preferred workflows and patterns automatically!

The system is ready to use. Start filling out the project_context/ files with 
your project information, and Bob will automatically use that knowledge when 
you give tasks.


================================================================================
EXAMPLE WORKFLOW
================================================================================

WITHOUT CONTEXT SYSTEM:
-----------------------
You: "Add user authentication"
Bob: "I'll add basic authentication. What pattern should I use?"
You: "Use JWT"
Bob: "What endpoints do you need?"
You: "Login and register"
Bob: "What's the workflow?"
... (many back-and-forth questions)


WITH CONTEXT SYSTEM:
--------------------
You: "Add user authentication"
Bob: [Automatically searches project_context/]
     [Finds: business_logic.md, architecture.md, api_documentation.md]
     [Reads all relevant context]
     "Based on your JWT architecture and auth workflow,
      I'll implement authentication with:
      - /auth/register endpoint with email verification
      - /auth/login endpoint returning JWT tokens
      - Password hashing with bcrypt
      - Token refresh mechanism
      
      Proceeding with implementation..."
